How It Works — Data Flow
User opens browser
       ↓
SvelteKit frontend (localhost:5173)
       ↓
+layout.svelte checks auth store
       ↓
Not logged in? → /login page → Google Sign-In popup → Firebase Auth
       ↓
Logged in? → Firebase gives ID token (JWT)
       ↓
Dashboard loads → calls fetchPortfolio()
       ↓
portfolio.ts sends GET /api/portfolio with Bearer token in header
       ↓
Go API receives request → middleware/auth.go verifies token with Firebase Admin
       ↓
Valid? → handlers/portfolio.go reads Firestore → returns assets
       ↓
Portfolio store updates → Svelte reactivity → UI re-renders

API Routes
GET    /health                         → health check (public)

GET    /api/portfolio                  → get all assets (auth required)
POST   /api/portfolio/asset            → add new asset
PUT    /api/portfolio/asset/:id        → update asset
DELETE /api/portfolio/asset/:id        → delete asset

GET    /api/stocks/price/:symbol       → live NSE price (e.g. RELIANCE)
GET    /api/mutualfunds/nav/:code      → live MF NAV (e.g. 120503)
GET    /api/gold/rate                  → live gold rate per gram INR

Firestore Data Structure

  {uid}/
    assets/
      {assetId}/
        type: "STOCK" | "GOLD" | "MUTUAL_FUND" | "PROPERTY" | "FD" | "NPS"
        name: string
        symbol: string
        quantity: number
        buyPrice: number
        buyDate: string (YYYY-MM-DD)
        currentPrice: number
        currentValue: number
        returns: number
        returnsPc: number
        metadata: {
          // Gold
          purity?: string
          grams?: number
          // Property
          address?: string
          areaSqFt?: number
          rentalYield?: number
          // FD
          bankName?: string
          interestRate?: number
          maturityDate?: string
          maturityAmount?: number
          // NPS
          tier?: string
          fundManager?: string
        }

Security Model
Layer 1 → Firebase Auth       User must be logged in
Layer 2 → Firestore Rules      Users can only read/write their own uid
Layer 3 → Go JWT middleware    Every API call verifies Firebase token
Layer 4 → Go input validation  All request bodies validated before DB write
Layer 5 → Rust                 Memory-safe, no null pointers in calculations
Layer 6 → .gitignore           secrets/ and .env never committed
